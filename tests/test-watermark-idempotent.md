# 测试：水线幂等

验证 `state/watermark.json` 的增量去重机制：同一日志跑两次记录会话，第二次不重复产草稿；水线损坏时降级全量重读不崩。

## 测试 A：正常幂等

### 输入
`adapters/fixtures/zcode-sample.jsonl`（2 行）

### 执行
1. 删除 `state/watermark.json`（确保从零开始）。
2. 跑一次 save（用 zcode 适配器，只读这个样本文件）。
3. 记录产出草稿数 N1。
4. 再跑一次 save。
5. 记录产出草稿数 N2。

### 期望断言
- [ ] N1 > 0（第一次产出了草稿）
- [ ] **N2 == 0**（第二次因水线生效，不重复）
- [ ] `watermark.json` 里该文件的水线 == 2（读完 2 行）
- [ ] now.md 没有重复追加同 session 的指针（幂等）

## 测试 B：水线损坏降级

### 输入
同上样本。

### 执行
1. 人为把 `watermark.json` 里该文件的水线改成 **9999**（超过实际行数 2）。
2. 跑一次 save。

### 期望断言
- [ ] **不崩**（捕获到水线 > 实际行数的异常）
- [ ] 水线被重置为 0，全量重读
- [ ] 产出草稿数 == 测试 A 的 N1（全量重读 = 等价于第一次）
- [ ] `watermark.json` 里该文件水线恢复为 2

## 失败处理
任一断言为假 → 报告失败 + 实际值。
